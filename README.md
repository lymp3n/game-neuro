# TextQuest RPG

Мобильная текстовая MMORPG: клиент на React Native (Expo) и игровой сервер на Node.js
(Express + WebSocket). В комплекте — веб-конструктор мира («админка»), который меняет
контент игры (локации, мобов, переходы) **в реальном времени**.

## Структура

```
game_Cursor/
├── client/      — приложение (Expo / React Native, работает на iOS, Android и в web)
├── server/      — игровой сервер (REST + WebSocket), раздаёт админку и веб-игру
│   └── src/data/content.json   — живой контент мира (создаётся автоматически, в .gitignore)
├── admin/       — веб-конструктор мира (HTML + JS, без сборки)
├── scripts/     — служебные скрипты (free-port)
└── render.yaml  — конфиг бесплатного хостинга
```

## Быстрый старт (локально)

```bash
# 1. Установить зависимости (один раз)
npm run install:all

# 2. Запустить сервер (порт 3001 освобождается автоматически)
npm run server

# 3. В отдельном терминале — клиент
npm run client          # Expo: нажмите w для web, или отсканируйте QR в Expo Go
```

- Игра в браузере: `http://localhost:8081` (после `w` в Expo).
- **Админка (конструктор мира): `http://localhost:3001/admin`**

### Игра с телефона (та же Wi‑Fi сеть)

При старте сервер печатает адрес вида `EXPO_PUBLIC_API_URL=http://192.168.x.x:3001`.
Задайте его перед запуском клиента:

```powershell
$env:EXPO_PUBLIC_API_URL="http://192.168.x.x:3001"; npm run client
```

## Конструктор мира (админка)

Откройте `http://localhost:3001/admin`. Возможности:

- **Бестиарий** (слева): создавайте мобов с названием, **описанием** и характеристиками
  (уровень, HP, физ./маг. урон, броня, опыт, иконка).
- **Холст локаций**: блоки-локации можно **перетаскивать** мышью.
  - «Новая локация» — создать блок; у каждого блока — название, тип (город / локация /
    вход в подземелье) и **описание**.
  - «Переход» на блоке → кликните по другому блоку, чтобы провести связь. Время перехода
    (в секундах) задаётся в окне перехода, переход можно сделать двусторонним.
  - **Перетащите моба** из бестиария на локацию, чтобы заселить её (или задайте число
    мобов в окне локации).
- **«Сохранить и применить»** — изменения пишутся в `content.json` и сразу применяются
  к запущенному серверу: подключённые игроки получают обновлённый мир без перезапуска.

> Безопасность: если задать переменную окружения `ADMIN_TOKEN`, админка попросит токен.
> Без неё (локально) доступ открыт.

## Как бесплатно разместить в интернете

Самый простой способ — один сервис на **[Render](https://render.com)** (бесплатный план),
который раздаёт сразу **игру (`/`)**, **админку (`/admin`)** и **API (`/api`, `/ws`)**.

1. Репозиторий уже на GitHub: `https://github.com/lymp3n/game-neuro`.
2. Зарегистрируйтесь на render.com (бесплатно, можно через GitHub).
3. **New → Blueprint** → выберите этот репозиторий. Render прочитает `render.yaml`
   и сам соберёт проект (`buildCommand`) и запустит сервер (`startCommand`).
4. После деплоя:
   - Игра: `https://<имя>.onrender.com/`
   - Админка: `https://<имя>.onrender.com/admin`
5. Токен админки сгенерируется автоматически — найдите `ADMIN_TOKEN` в разделе
   **Environment** сервиса и введите его при первом входе в `/admin`.

Альтернативы с тем же подходом (Node-сервис + статика): **Railway**, **Fly.io**, **Koyeb**.
Шаги те же: подключить GitHub-репозиторий, указать build `npm install && npm --prefix server install && npm --prefix client install && npm --prefix client run build:web`,
start `node server/src/index.js`.

> Замечание про бесплатный план Render: после простоя сервис «засыпает» и первый запрос
> идёт ~30 сек. Файл `content.json` на бесплатном плане не сохраняется между перезапусками
> (эфемерный диск) — правки из админки живут до перезапуска. Чтобы правки были постоянными,
> подключите платный Persistent Disk либо коммитьте `content.json` в репозиторий.

## Полезные команды

```bash
npm run server      # сервер + админка (с авто-освобождением порта 3001)
npm run server:stop # просто освободить порт 3001
npm run client      # Expo dev (телефон/web)
npm run build:web   # собрать веб-версию игры в client/dist (её раздаёт сервер на /)
npm run dev         # сервер и клиент вместе
```
